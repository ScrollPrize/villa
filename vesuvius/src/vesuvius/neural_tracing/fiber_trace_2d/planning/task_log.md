# Task Log: Cached Fused Augmentation Maps For Line And CP Warp

## Implementation Notes

- `StripAugmentTransform` now caches shape/parameter/device-dependent affine
  constants, centers, shear determinant, scale, and smooth-offset controls at
  construction.
- Smooth-offset controls are generated by a stateless vectorized deterministic
  tensor hash and are no longer produced by constructing a `torch.Generator` in
  hot mapping calls.
- Coordinate augmentation and line/CP mapping can share one fused transform
  object in `build_strip_patch_from_source`.
- `_line_and_cp_xy_for_params` concatenates source line rows and CP into one
  tensor, calls `source_to_output_points` once, then splits the mapped line and
  CP outputs.
- `build_sample` caches transforms and transformed line/CP pairs by
  augmentation params, so identical params across strip-z offsets reuse the
  same mapping.
- Added focused tests for cached smooth controls, single transform reuse for
  coordinate+line mapping, batched line+CP mapping, and strip-z line/CP reuse.
